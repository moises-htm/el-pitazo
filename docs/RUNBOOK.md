## Required Environment Variables

Set these in **Vercel → Project → Settings → Environment Variables**.

| Variable | Required | Notes |
|----------|----------|-------|
| `DATABASE_URL` | ✅ | Neon PostgreSQL. Include `?sslmode=require` |
| `JWT_SECRET` | ✅ | Min 32 chars. Generate: `openssl rand -base64 32` |
| `MP_SECRET_KEY` | ✅ | MercadoPago secret access token |
| `MP_PUBLIC_KEY` | ✅ | MercadoPago public key |
| `MP_WEBHOOK_SECRET` | ✅ | From MP Dashboard → App → Webhooks → Show secret |
| `BLOB_READ_WRITE_TOKEN` | ✅ | Vercel Blob → Store → Token |
| `NEXT_PUBLIC_BASE_URL` | ✅ | `https://elpitazo.app` (no trailing slash) |
| `NEXT_PUBLIC_SENTRY_DSN` | ✅ | sentry.io → Project → Settings → DSN |
| `SENTRY_ORG` | ✅ | Sentry org slug |
| `SENTRY_PROJECT` | ✅ | Sentry project slug |
| `SENTRY_AUTH_TOKEN` | ✅ | sentry.io → Settings → Auth Tokens |
| `VAPID_PUBLIC_KEY` | ⚠️ | Web push. Generate: `npx web-push generate-vapid-keys` |
| `VAPID_PRIVATE_KEY` | ⚠️ | Web push (same command as above) |
| `VAPID_SUBJECT` | ⚠️ | `mailto:admin@elpitazo.app` |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | ⚠️ | Same value as `VAPID_PUBLIC_KEY` |
| `CRON_SECRET` | ⚠️ | Protects `/api/cron/*` — set a random string |
| `NEXTAUTH_SECRET` | ⚠️ | NextAuth. Generate: `openssl rand -base64 32` |
| `NEXTAUTH_URL` | ⚠️ | `https://elpitazo.app` |
| `NEXT_PUBLIC_GOOGLE_MAPS_KEY` | optional | Maps Static API key |
| `GOOGLE_CLIENT_ID` | optional | Google OAuth |
| `GOOGLE_CLIENT_SECRET` | optional | Google OAuth |

## Deploy

Deployments trigger automatically on push to `main`.

```bash
git push origin main
```

Build pipeline: `prisma generate → prisma migrate deploy → next build`

---

## Rollback

```bash
# Vercel CLI
vercel rollback

# Or: Vercel Dashboard → Deployments → pick previous → ⋯ → Promote to Production
```

```bash
curl https://elpitazo.app/api/health
```

## Database

- **Provider:** Neon (neon.tech) — free tier has auto-pause after 5 min idle
- **Connection string:** Vercel → Environment Variables → `DATABASE_URL`
- **Backups:** Neon Dashboard → Project → Branches → Restore (Scale plan has PITR)
- **Connect manually:**
  ```bash
  psql $DATABASE_URL
  ```
- **Run migration manually (if build missed it):**
  ```bash
  npx prisma migrate deploy
  ```
- **Studio (local):**
  ```bash
  npm run db:studio
  ```

## Cron Jobs

| Schedule (UTC) | Route | Purpose |
|----------------|-------|---------|
| 09:00 daily | `/api/cron/match-reminders` | Push reminders for next-day matches |

Trigger manually:
```bash
curl -H "x-cron-secret: $CRON_SECRET" https://elpitazo.app/api/cron/match-reminders
```

## MercadoPago Webhooks

1. MP Dashboard → Your Application → Webhooks  
2. Add URL: `https://elpitazo.app/api/payments/mercadopago/webhook`  
3. Select events: **payment**  
4. Copy the webhook secret → Vercel env var `MP_WEBHOOK_SECRET`

## Incident Response

1. **Check health:** `curl https://elpitazo.app/api/health`
2. **If db-error:** Check Neon dashboard for instance status / idle auto-pause
3. **Check Sentry** for error spike
4. **Check Vercel Logs** for 5xx patterns
5. **If payments broken:** Check MP dashboard → API Status page
6. **Rollback** if error rate > 5% and root cause unclear (see Rollback above)
7. **Hot fix:** create branch → push → merge → auto-deploy takes ~2 min

## Common Tasks

### Reset a user password (DB)
```sql
-- Generate new hash first: node -e "require('bcryptjs').hash('newpass',12).then(console.log)"
UPDATE users SET password = '<bcrypt_hash>' WHERE email = 'user@example.com';
```

### Manually mark a team payment as paid (cash)
```
POST /api/payments/cash/confirm
Authorization: Bearer <organizer_jwt>
{ "tournamentId": "...", "teamId": "..." }
```

### Promote to production (if auto-deploy is off)
```bash
vercel --prod
```
